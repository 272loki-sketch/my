# Replit 部署指南（Agent 版）

这份文档记录本项目在 Replit 上从零部署的完整流程，以及本次实际部署中遇到的问题和对应解决办法。下次让 Agent 部署时，直接让它按本文执行即可。

项目源码在仓库的 `new-api/` 子目录中。Replit 工作区根目录可能还包含模板文件，例如 `package.json`、`pnpm-lock.yaml`、`lib/`、`scripts/` 等，这些不是后端项目本体。

---

## 1. 最终可用方案

Replit 环境的限制：

- 不能使用 `sudo`
- 不能使用 `apt install`
- 没有 `systemd`
- Nix channel 较旧，默认 `pkgs.go` 可能只有 Go 1.16
- Nix channel 没有 `pkgs.bun`
- 前端在 Replit 上直接 `bun run build` 可能因 Semi UI 依赖提升问题失败

最终成功方案：

- Go 手动下载 `go1.25.1.linux-amd64.tar.gz`
- Bun 手动安装到 `~/.bun/bin`
- `replit.nix` 只安装 `gcc` 和 `unzip`
- Windows 本地提前构建 `new-api/web/dist`，并提交到 GitHub
- Replit 上跳过前端构建，只构建后端
- 后端二进制命名为 `new-api-bin`，避免和 `new-api/` 目录冲突

---

## 2. 推荐仓库结构

当前 GitHub 仓库结构大致是：

```text
repo-root/
├── new-api/
│   ├── go.mod
│   ├── main.go
│   ├── web/
│   │   ├── package.json
│   │   ├── bun.lock
│   │   └── dist/          # 必须提交，Replit 跳过前端构建依赖它
│   └── ...
├── DEPLOY.md
├── REPLIT_DEPLOY_GUIDE.md
└── ...
```

如果 Agent 要把 `new-api/` 移到 Replit 根目录也可以，但要注意 `git pull` 后新增文件仍可能落回 `new-api/` 子目录，容易混乱。因此后续建议直接在 `~/workspace/new-api` 中运行项目，不再移动目录。

## 3. 完整部署流程（从零开始，推荐照抄）

这一节是最终验证过的完整流程。除非有明确原因，不要改动目录结构，不要把 `new-api/` 移到根目录。

### 3.1 维护者先在 Windows 提交预构建前端

Replit 上不要构建前端。维护者在 Windows 本地执行：

```powershell
cd D:\newapi\new-api
& ".\build-frontend.ps1"

# web/dist 被 .gitignore 忽略时必须 -f
git add -f web/dist web/bun.lock
git commit -m "add prebuilt frontend assets"
git push origin main
```

确认 GitHub 上存在：

```text
new-api/web/dist/index.html
new-api/web/bun.lock
```

如果 GitHub 推送 `web/dist` 因网络或大文件传输失败，可以改用手动压缩包方案：

```powershell
cd D:\newapi
Compress-Archive -Path .\new-api\web\dist\* -DestinationPath .\new-api-web-dist.zip -Force
```

然后把仓库根目录的 `new-api-web-dist.zip` 上传给 Replit Agent，让它在 Replit Shell 执行：

```bash
cd ~/workspace
rm -rf ~/workspace/new-api/web/dist
mkdir -p ~/workspace/new-api/web/dist
unzip -o ~/workspace/new-api-web-dist.zip -d ~/workspace/new-api/web/dist
ls ~/workspace/new-api/web/dist/index.html
```

注意：压缩包内应直接包含 `index.html` 和 `assets/`，不要多套一层 `dist/` 目录。本文档旁边的 `new-api-web-dist.zip` 就是按这个格式生成的。

### 3.2 Replit 清空模板并拉取仓库

在 Replit Shell 执行：

```bash
cd ~/workspace
rm -rf ./* ./.[!.]* 2>/dev/null
git clone https://github.com/272loki-sketch/my.git .
```

确认源码位置：

```bash
ls ~/workspace/new-api/go.mod ~/workspace/new-api/web/package.json ~/workspace/new-api/web/dist/index.html
```

如果 `web/dist/index.html` 不存在，先停止部署，回到 Windows 提交预构建前端。

### 3.3 写入 Replit 配置

```bash
cat > ~/workspace/replit.nix << 'EOF'
{ pkgs }: {
  deps = [
    pkgs.gcc
    pkgs.unzip
  ];
}
EOF

cat > ~/workspace/.replit << 'EOF'
run = "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"

[deployment]
router = "application"
deploymentTarget = "gce"
build = ["sh", "-c", "cd new-api && test -f web/dist/index.html && go build -o new-api-bin ."]
run = ["sh", "-c", "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"]

[env]
PATH = "/home/runner/go/bin:/home/runner/.bun/bin:${REPLIT_ENV_PATH}"
GOPATH = "/home/runner/gopath"
GOPROXY = "https://goproxy.cn,direct"
CGO_ENABLED = "0"

[[ports]]
localPort = 3000
externalPort = 80
EOF
```

等 Replit Nix 环境刷新完成。如果出现 `Failed to compile new environment`，执行：

```bash
cat /run/replit/env/error
```

确认没有 `attribute 'bun' missing`。本文最终配置中不使用 `pkgs.bun`。

### 3.4 手动安装 Go 1.25.1 与 Bun

```bash
cd ~
rm -rf ~/go
curl -Lo go1.25.1.linux-amd64.tar.gz https://go.dev/dl/go1.25.1.linux-amd64.tar.gz
tar -xzf go1.25.1.linux-amd64.tar.gz
rm go1.25.1.linux-amd64.tar.gz

curl -fsSL https://bun.sh/install | bash

export PATH="$HOME/go/bin:$HOME/.bun/bin:$PATH"
export GOPATH="$HOME/gopath"
export GOPROXY="https://goproxy.cn,direct"

go version
bun --version
```

期望：

```text
go version go1.25.1 linux/amd64
1.3.x
```

如果 `curl -fsSL https://bun.sh/install | bash` 报：

```text
error: unzip is required to install bun
```

说明 `replit.nix` 没有生效或缺少 `pkgs.unzip`，回到 3.3 重写并等待环境刷新。

### 3.5 设置 Replit Secrets

在 Replit 左侧 `Secrets`（锁形图标）添加：

```text
SESSION_SECRET = 随机字符串
SQLITE_PATH = ./one-api.db?_busy_timeout=30000
PORT = 3000
# 可选：风控可信用户 ID，命中的用户不会进入可疑用户列表
TRUSTED_USER_IDS = 1,2,3
```

生成随机字符串：

```bash
openssl rand -hex 32
```

如启用 Discord 校验，添加：

```text
DISCORD_REQUIRED_GUILD_ID = 你的 Discord 服务器 ID
DISCORD_REQUIRED_ROLE_IDS = 身份组ID1,身份组ID2,身份组ID3
# 可选：可信 Discord 用户 ID，命中后跳过服务器和身份组校验
DISCORD_TRUSTED_USER_IDS = Discord用户ID1,Discord用户ID2
```

### 3.6 构建后端

```bash
cd ~/workspace/new-api

# 确认前端产物存在，后端 embed 依赖它
ls web/dist/index.html

# 确认 Go 版本
export PATH="$HOME/go/bin:$HOME/.bun/bin:$PATH"
export GOPROXY="https://goproxy.cn,direct"
go version

# 构建，输出名不要叫 new-api，避免和目录冲突
go build -o new-api-bin .
chmod +x new-api-bin
mkdir -p logs
```

### 3.7 手动启动测试

```bash
cd ~/workspace/new-api
./new-api-bin --port 3000 --log-dir ./logs
```

新开一个 Shell，验证：

```bash
curl http://127.0.0.1:3000/api/status
```

返回 JSON 即成功。

如果输出中有：

```text

```

表示还未初始化，打开网页后按初始化页面创建管理员即可。

### 3.8 使用 Run 按钮启动

手动测试成功后：

1. 在运行日志的 Shell 中按 `Ctrl+C` 停止服务
2. 点击 Replit 顶部 `Run`
3. 到 `Ports` 或 `Webview` 打开 3000 端口

如果 Webview 显示：

```text
the artifact is running, but the preview isn't ready
```

先执行：

```bash

```

若能返回 JSON，说明服务已运行，只是 Replit 预览没及时识别。到 `Ports` 手动打开 3000。

预览能打开后再发布。发布后进入后台：

- 系统设置中的风控黑名单：按需填写 Discord 用户 ID 黑名单和 IP/CIDR 黑名单


### 3.10 SQLite 数据库导入/导出

后台已提供 SQLite 数据库迁移功能，入口在：

```text
系统设置 -> 性能设置 -> 数据库备份
```

可用操作：

- `导出 SQLite 数据库`：下载当前 SQLite 主数据库快照。
- `导入 SQLite 数据库`：上传导出的 `.db` 备份并替换当前 SQLite 数据库。

安全限制：

- 仅超级管理员可用。
- 操作前必须完成 2FA 或 Passkey 二次验证。
- 导入文件会校验是否为有效 SQLite，并检查 `users`、`options` 等关键表。
- 导入前会把当前数据库备份为 `one-api.db.before-import-时间.bak`。
- 导入成功后服务会自动停止，需要手动重新启动服务。

Replit 导入后的重启方式：

```bash
cd ~/workspace/new-api
./new-api-bin --port 3000 --log-dir ./logs
```

或直接点击 Replit 顶部 `Run`。如果是 Deployment 环境，导入后需要重新 Deploy 或重启服务。

注意：数据库备份文件包含用户、Token、渠道、模型价格、充值记录、日志等敏感数据。不要提交到 GitHub，不要上传到公开网盘或聊天群。仓库 `.gitignore` 已忽略 `*.db` 和 `*.db-journal`。

### 3.11 风控、蜜罐模型和多密钥测活

后台风控入口：

```text
系统设置 -> 系统设置 -> 风控设置
```

支持：

- `可信用户 ID`：减少可疑用户误报，不影响限速、额度和日志。
- `Discord 用户 ID 黑名单`：命中后禁止 Discord 登录/注册；已注册用户会自动封禁。
- `IP 黑名单`：支持单 IP 和 CIDR，例如 `1.2.3.4`、`1.2.3.0/24`。命中后禁止注册，已登录/API 调用用户会自动封禁。

也可以通过 Replit Secrets 配置：

```text
RISK_BLOCKED_DISCORD_USER_IDS = Discord用户ID1,Discord用户ID2
RISK_BLOCKED_IPS = 1.2.3.4,1.2.3.0/24
```

蜜罐模型入口：

```text
系统设置 -> 模型相关设置 -> 全局设置
```

支持：

- `假模型 / 蜜罐模型列表`：例如 `gpt10`、`deepseekv10`、`claude opus10`。
- `假模型返回内容`：用户调用假模型时直接返回该内容，支持普通请求和流式请求，默认 `baka`。

说明：假模型会出现在 `/v1/models` 和渠道获取模型列表中，方便 Chatbox/酒馆侧选择；真实调用不会请求上游。

多密钥测活入口：

```text
渠道管理 -> 多密钥管理 -> 测试全部密钥
```

行为：

- 不会自动删除任何 key。
- 测活成功会解除自动禁用状态。
- 测活失败会标记为自动禁用。
- 手动禁用的 key 会参与测试但不会自动改状态，避免覆盖人工判断。

## 6. 不要把 Nix 表达式直接粘到 Shell

错误示例：

```bash
{ pkgs }: {
  deps = [
    pkgs.gcc
    pkgs.unzip
  ];
}
```

这样会让 Shell 把 `pkgs` 当成命令执行，出现：

```text
pkgs: command not installed
bash: deps: command not found
```

正确做法是写入 `replit.nix` 文件：

```bash
cat > ~/workspace/replit.nix << 'EOF'
{ pkgs }: {
  deps = [
    pkgs.gcc
    pkgs.unzip
  ];
}
EOF
```

`unzip` 是安装 Bun 时必须的，否则会报：

```text
error: unzip is required to install bun
```

## 11. Replit 前端构建失败说明

Replit 上如果尝试：

```bash
cd ~/workspace/new-api/web
bun install
NODE_OPTIONS=--max-old-space-size=4096 bun run build
```

可能报错：

```text
Missing "./dist/css/semi.css" specifier in "@douyinfe/semi-ui" package
```

或：

```text
Can't find stylesheet to import.
@import "~@douyinfe/semi-theme-default/scss/index.scss";
```

原因：

- Replit 上 Bun 版本、平台、依赖提升策略与 Windows 不一致
- Semi UI/Semi Icons 的嵌套依赖解析异常
- 即使提交 `bun.lock`，在 Replit 上仍可能因嵌套 `node_modules` 解析失败

处理原则：

- 不在 Replit 上构建前端
- 在 Windows 本地构建 `web/dist`
- 将 `web/dist` 强制提交到 GitHub
- Replit 只构建 Go 后端

## 12. 后端构建常见错误

### 12.1 Go 版本太低

错误：

```text
golang.org/x/sync@v0.20.0 requires go >= 1.25.0 (running go 1.24.2)
```

解决：安装 Go 1.25.1：

```bash
cd ~
rm -rf ~/go
curl -Lo go1.25.1.linux-amd64.tar.gz https://go.dev/dl/go1.25.1.linux-amd64.tar.gz
tar -xzf go1.25.1.linux-amd64.tar.gz
rm go1.25.1.linux-amd64.tar.gz
export PATH="$HOME/go/bin:$HOME/.bun/bin:$PATH"
go version
```

### 12.2 缺少 `web/dist`

错误：

```text
main.go:37:12: pattern web/dist: no matching files found
```

原因：后端使用 `embed` 嵌入前端静态文件，但 `web/dist` 不存在。

解决：在 Windows 本地构建并提交 `web/dist`，然后 Replit `git pull`。

### 12.3 二进制名称与目录冲突

错误：

```text
bash: ./new-api: Is a directory
```

原因：仓库中有 `new-api/` 目录，`go build -o new-api .` 输出名冲突。

解决：使用：

```bash
go build -o new-api-bin .
./new-api-bin --port 3000 --log-dir ./logs
```

如果预览显示：

```text
the artifact is running, but the preview isn't ready
```

通常是 Replit 预览未识别端口。建议：

1. 停止手动运行的服务：`Ctrl+C` 或 `pkill -f new-api-bin || true`
2. 确认 `.replit` 配置正确
3. 点击 Replit 顶部 `Run`
4. 从 `Ports` 打开 3000 端口

不要在预览没通之前直接发布。先确认本地端口和 Webview 正常。

### 13.1 发布时报 `Could not find run command`

这个错误表示 Replit Deployment 没有在 `.replit` 里找到部署运行命令。只写顶层：

```toml
run = "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"
```

有时只能让工作区的 Run 按钮正常，但 Deployment 仍然找不到命令。必须同时写 `[deployment]` 里的 `build` 和 `run`：

```toml
run = "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"

[deployment]
router = "application"
deploymentTarget = "gce"
build = ["sh", "-c", "cd new-api && test -f web/dist/index.html && go build -o new-api-bin ."]
run = ["sh", "-c", "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"]
```

如果项目已经被移动到 `~/workspace` 根目录，则使用：

```toml
run = "./new-api-bin --port 3000 --log-dir ./logs"

[deployment]
router = "application"
deploymentTarget = "gce"
build = ["sh", "-c", "test -f web/dist/index.html && go build -o new-api-bin ."]
run = ["sh", "-c", "./new-api-bin --port 3000 --log-dir ./logs"]
```

保存后重新发布即可。

首次打开网页后：

- 如果看到初始化页面，按页面创建管理员
- 如果看到登录页，默认账号通常为：

```text
root
```

首次登录后立即修改密码。

进入后台后，将网站地址改为 Replit 发布域名：


API Base URL：

```text

```

Discord OAuth 回调地址：

```text
https://你的应用名.replit.app/oauth/discord
```

Discord 可信用户白名单：

```text
后台 -> 系统设置 -> 配置 Discord OAuth -> Discord 用户 ID（可信白名单）
```

填写示例：

```text
345678901234567890,456789012345678901
```

命中这些 Discord 用户 ID 的账号可以只通过 Discord 登录注册，不需要属于指定服务器，也不需要拥有指定身份组。这个白名单适合允许可信朋友注册，例如朋友不在你的 Discord 服务器内。也可以通过 Replit Secrets 设置 `DISCORD_TRUSTED_USER_IDS=用户ID1,用户ID2`。

可信用户 ID 白名单：

```text
后台 -> 系统设置 -> 风控设置 -> 可信用户 ID
```

填写示例：

```text
1,2,3
```

命中这些 ID 的用户不会进入「可疑用户」列表。这个白名单只影响可疑用户列表，不影响限速、额度、日志记录和请求头审计。也可以通过 Replit Secrets 设置 `TRUSTED_USER_IDS=1,2,3` 作为默认配置。

## 15. 如果已经把 `new-api/` 移到 workspace 根目录

这次实际部署过程中曾经执行过：

```bash
shopt -s dotglob
mv new-api/* .
shopt -u dotglob
rmdir new-api
```

这会导致源码位于 `~/workspace` 根目录，而后续 `git pull` 又可能把新提交拉到 `~/workspace/new-api/`，例如：

```text
~/workspace/new-api/web/bun.lock
~/workspace/new-api/web/dist
```

如果已经处于这种混合状态，可用以下命令修复：

```bash
cd ~/workspace

# 如果 git pull 后新增的 dist 在子目录，把它复制到当前实际源码目录
if [ -d ~/workspace/new-api/web/dist ]; then
  rm -rf ~/workspace/web/dist
  cp -r ~/workspace/new-api/web/dist ~/workspace/web/dist
fi

if [ -f ~/workspace/new-api/web/bun.lock ]; then
  cp ~/workspace/new-api/web/bun.lock ~/workspace/web/bun.lock
fi

# 使用根目录源码构建
export PATH="$HOME/go/bin:$HOME/.bun/bin:$PATH"
export GOPROXY="https://goproxy.cn,direct"
ls web/dist/index.html
go build -o new-api-bin .
chmod +x new-api-bin
mkdir -p logs
./new-api-bin --port 3000 --log-dir ./logs
```

对应 `.replit`：

```bash
cat > ~/workspace/.replit << 'EOF'
run = "./new-api-bin --port 3000 --log-dir ./logs"

[deployment]
router = "application"
deploymentTarget = "gce"
build = ["sh", "-c", "test -f web/dist/index.html && go build -o new-api-bin ."]
run = ["sh", "-c", "./new-api-bin --port 3000 --log-dir ./logs"]

[env]
PATH = "/home/runner/go/bin:/home/runner/.bun/bin:${REPLIT_ENV_PATH}"
GOPATH = "/home/runner/gopath"
GOPROXY = "https://goproxy.cn,direct"
CGO_ENABLED = "0"

[[ports]]
localPort = 3000
externalPort = 80
EOF
```

长期建议：新部署不要移动 `new-api/`，直接在 `~/workspace/new-api` 中构建运行。

## 16. 成功部署后的最短命令清单

假设源码保持在 `~/workspace/new-api`，并且 GitHub 已包含 `new-api/web/dist`：

```bash
cd ~/workspace
cat > replit.nix << 'EOF'
{ pkgs }: {
  deps = [
    pkgs.gcc
    pkgs.unzip
  ];
}
EOF

cat > .replit << 'EOF'
run = "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"

[deployment]
router = "application"
deploymentTarget = "gce"
build = ["sh", "-c", "cd new-api && test -f web/dist/index.html && go build -o new-api-bin ."]
run = ["sh", "-c", "cd new-api && ./new-api-bin --port 3000 --log-dir ./logs"]

[env]
PATH = "/home/runner/go/bin:/home/runner/.bun/bin:${REPLIT_ENV_PATH}"
GOPATH = "/home/runner/gopath"
GOPROXY = "https://goproxy.cn,direct"
CGO_ENABLED = "0"

[[ports]]
localPort = 3000
externalPort = 80
EOF

cd ~
rm -rf ~/go
curl -Lo go1.25.1.linux-amd64.tar.gz https://go.dev/dl/go1.25.1.linux-amd64.tar.gz
tar -xzf go1.25.1.linux-amd64.tar.gz
rm go1.25.1.linux-amd64.tar.gz

curl -fsSL https://bun.sh/install | bash

export PATH="$HOME/go/bin:$HOME/.bun/bin:$PATH"
export GOPROXY="https://goproxy.cn,direct"

cd ~/workspace/new-api
ls web/dist/index.html
go build -o new-api-bin .
chmod +x new-api-bin
mkdir -p logs
./new-api-bin --port 3000 --log-dir ./logs
```

新开 Shell 验证：

```bash
curl http://127.0.0.1:3000/api/status
```

返回 JSON 即部署成功。
